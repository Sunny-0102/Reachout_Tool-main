One-time Setup
MailBlast can send through Gmail or Outlook / Microsoft 365. Add either or both client IDs below so you can use your Google account, personal Microsoft account, or your college Microsoft 365 mailbox from the same tool.
1
For Gmail: go to Google Cloud Console → create a project → enable Gmail API → create a Web application OAuth client.
2

3
In Google Auth Platform → Audience, if the app is External and still in Testing, add every Gmail address you want to use under Test users. Otherwise Google will show Error 403: access_denied.
4
For Outlook / Microsoft 365: go to Microsoft Entra / Azure Portal → App registrations → create an app → set supported account types to include personal and, if needed, work or school accounts.
5
Under Authentication, add a Single-page application redirect URI using this exact page URL:
6
For Microsoft permissions, add delegated Mail.Send and User.Read. Some school tenants may also require admin consent.
If another Gmail account gets blocked during sign-in, open the Google Cloud project for this client ID and add that Gmail under Google Auth Platform → Audience → Test users, or move the OAuth app to production after Google verification.
For a college Microsoft account, this usually only works if your college email is hosted on Microsoft 365 and the tenant allows Mail.Send. If sign-in works but sending is blocked, your school admin likely needs to grant consent for the app.
